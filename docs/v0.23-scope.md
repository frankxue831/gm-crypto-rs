# v0.23 scope — pre-1.0 re-audit remediation

**Theme:** fix the BLOCKERs + genuine should-fixes surfaced by the v1.0 pre-publish
re-audit (`docs/v1.0-reaudit.md`), so the deliberate `1.0.0` publish freezes a *correct,
minimal, honestly-documented* contract. **Non-publishing** (workspace stays `0.16.0`; the
breaking API/ABI changes ship **with** 1.0, never reaching a published 0.x crate — the
v0.14/v0.17–v0.22 precedent). Input: `docs/v1.0-reaudit.md` §2–§5.

## 1. Context

The re-audit (two independent model reviews — Codex `gpt-5.5` + Grok — each finding
adversarially source-verified) returned **NO-GO as-is**: the **core primitives are sound**
(mutually-confirmed CT/correctness non-findings — `docs/v1.0-reaudit.md` §3), but there are
**2 API BLOCKERs** + a set of API-finality and zeroize-on-failure / spec-ceiling / doc
should-fixes that become irreversible or harder to fix after 1.0. This cycle remediates
them, then a **clean re-run of the multi-model review on the diff** is the gate to publish.

## 2. Hard contract (non-negotiable)

- **Non-publishing.** No version bump (stays `0.16.0`), no tag, no `cargo publish`. The
  breaking API/ABI *shape* changes ship with the deliberate 1.0 cut.
- **Runtime output byte-identical** for everything except the *deliberately* changed
  signatures (B-1 GCM fallibility): full KAT + gmssl interop **11/11** must stay green;
  the SM2 DER signature + ciphertext wire bytes, SM4 mode outputs, PKCS#8/SPKI fixtures
  are all unchanged. Zeroization changes are not observable in output.
- **All standing invariants preserved:** `unsafe_code = "forbid"` on core; constant-time
  discipline (no new `==`/`if`/`bool` on secrets — and B-7 *removes* one); single
  `Failed`/`None` failure-mode invariant (B-1's new `None`/`Err` is a *length-range* reject,
  not a failure-mode distinction); MSRV 1.85 / edition 2024; `Cargo.lock` gitignored.
- **Each workstream gets a `codex exec --sandbox read-only` milestone review** (the
  established cadence). Branch + PR; never direct to `main`.

## 3. Decisions (Q23.x) — recommended resolutions for Codex confirmation + maintainer scope approval

Forks per `docs/v1.0-reaudit.md` §5. **Bold = recommended.**

- **Q23.1 — A-2 `ProjectivePoint` depth.** **Reshape the high-level path to speak
  keys/bytes:** `Sm2PrivateKey::public_key() -> Sm2PublicKey`; `Sm2PublicKey::from_sec1_bytes`
  (already on-curve-checked) is the public constructor; `#[doc(hidden)]`
  `Sm2PublicKey::{from_point, point}`, `From<ProjectivePoint> for Sm2PublicKey`, and the
  bare `ProjectivePoint` arithmetic (`add/double/neg`; `generator`/`identity`/`is_identity`
  kept as needed for internal/dev use, doc-hidden). `ProjectivePoint` stays a public
  *opaque* type (decouple-only posture preserved) but is no longer an unchecked key
  constructor or an EC-arithmetic API. *Alternative (rejected unless Codex prefers): keep
  the point methods public + only add the on-curve check to `from_point`.*
- **Q23.2 — A-1 C ABI contract.** **Make the AEAD + XTS FFI symbols always-on in
  `gmcrypto-c`:** drop the forwarding `sm4-aead`/`sm4-xts` cargo features on the C shim so
  the committed `gmcrypto.h` == the default `cargo build -p gmcrypto-c` (one complete,
  permanent ABI). `gmcrypto-core` keeps its feature gates (Rust callers opt in); the C
  shim (std-OK integration layer) always compiles every mode. `regen-header` no longer
  needs to "imply" anything. *Alternative (rejected unless Codex prefers): `#ifdef`-guard
  the header + ship a feature→symbol map (fragile for C consumers).*
- **Q23.3 — B-1 single-shot GCM fallibility.** **Make `mode_gcm::{encrypt,
  encrypt_with_tag_len}` fallible** (return the workspace `Result`/`Option`, rejecting
  `> 2^36 − 32` bytes), matching the streaming poison + `decrypt`'s `Option`. The C FFI
  GCM encrypt already returns an error code, so the ABI is unaffected. *Lighter alternative
  (Codex to weigh): keep infallible + document the ceiling as a caller contract.* The
  matching `decrypt` length-guard is additive (already `Option`).
- **Q23.4 — A-3 `rand_core` bound.** **Accept-and-document; do NOT decouple.** `rand_core`
  **is** the ecosystem RNG-interop point (unlike `crypto-bigint`, a numeric impl detail);
  naming `CryptoRng + Rng` is the idiomatic, interop-preserving choice (RustCrypto-wide).
  A custom RNG trait would break interop for net-negative benefit. Document the deliberate,
  ecosystem-aligned coupling (mirroring the `crypto-bigint-scalar` escape-hatch reasoning).
  **This is the key fork to confirm.**
- **Q23.5 — A-4/A-5 surface trim.** **`#[doc(hidden)]` `asn1::{reader,writer,oid}` +
  `traits::{Hash,Mac,BlockCipher}`** (keep `pub` for internal cross-module/dev-crate use —
  the Group-A pattern). Keep `asn1::{encode_sig,decode_sig}` + `asn1::ciphertext::Sm2Ciphertext`
  public (the v0.22 wire types). Regenerate the cargo-public-api baseline.
- **Q23.6 — zeroization + ceiling fixes (not forks, confirm approach).** CCM
  `tentative_pt.zeroize()` on the tag-fail arm (B-3); enable `crypto-bigint`'s `zeroize`
  feature and wipe the SM2 sign nonce `k` + scalar intermediates + RNG buffer (B-4); add
  `HmacSm3` `Drop` + zeroize to honor its doc (B-2); SM2 KDF `u32` length guard (B-5,
  `encrypt` already fallible). All defense-in-depth; no output change.
- **Q23.7 — B-7 nonce sampling.** **Make `sample_nonzero_scalar`'s rejection CT** (drop the
  `if bool::from(..)` in favor of a fixed-shape masked-select retry, or a CT loop) so the
  documented "no `bool` on secrets" invariant holds literally. *Alternative: explicitly
  carve it out in the CT-posture docs.* (Reject prob ≈ `2^-224`; either is sound.)
- **Q23.8 — docs/disclosure (C-1 + D-1 + B-6/8/9/10).** Fix `gmcrypto-core` `description`
  (add SM4); sharpen README audit-status to "no *third-party* audit; assurance is internal
  (multi-model re-audit + KAT + gmssl interop + 18 fuzz + dudect)"; add SECURITY notes for
  the streaming-CBC contract, short-PT encrypt timing, and the "CT-until-the-validity-bit"
  posture; keep GM/T-certification-out-of-scope.
- **Q23.9 — publish posture.** **Non-publishing** (stays `0.16.0`; ships with 1.0).
