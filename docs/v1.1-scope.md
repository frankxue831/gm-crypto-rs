## Q-list (locked decisions)

- **Q1.1 Direction** ‚Äî SM2 key exchange (GM/T 0003.3 ‚â° GB/T 32918.3-2016) is the
  v1.1 capability. It completes the SM2 family (0003.2 sign + 0003.4/.5 encrypt
  already ship) and is the prerequisite for an eventual TLCP capability. Decided
  via a Codex consult + market scan (the primitive space is saturated; the gap is
  the protocol layer).
- **Q1.2 Key confirmation** ‚Äî **included**: full authenticated AKE with the
  `S_A`/`S_B` SM3 confirmation tags. The typestate makes the agreed key
  unreachable without passing through `confirm`/`finish`.
- **Q1.3 API shape** ‚Äî role state-machines (`Sm2KxInitiator` ‚Üí `‚Ä¶Waiting` ‚Üí
  key; `Sm2KxResponder` ‚Üí `‚Ä¶Waiting` ‚Üí key), mirroring the streaming-cipher
  idiom: advancing consumes `self`, so an ephemeral cannot be reused and the
  key cannot be fetched before confirmation.
- **Q1.4 Packaging** ‚Äî opt-in feature `sm2-key-exchange = []` on
  `gmcrypto-core`. Pure-core: **no new dependency** (reuses the SM2 curve
  arithmetic, the fixed-budget masked nonce sampler, the SM3 KDF, `compute_z`,
  and the SEC1 on-curve validation). Default-features build byte-identical.
- **Q1.5 C FFI** ‚Äî **deferred to v1.2** (the core-in-vN / FFI-in-vN+1 cadence;
  the v0.12‚Üív0.13 XTS precedent). `gmcrypto-c` is unchanged this cycle but
  bumps in lockstep (`=1.1.0` pins).
- **Q1.6 KAT source** ‚Äî the GB/T 32918.5-2017 / GM/T 0003.5 recommended-curve
  worked example (NOT the GB/T 32918.3 test-curve Annex ‚Äî wrong curve for this
  library). See `docs/v1.1-sm2kx-kat-sourcing.md`.
- **Q1.7 All-zero-K reject** ‚Äî `K = 0^klen` ‚Üí `Failed` is **deliberate
  hardening** beyond GB/T 32918.3: it is GB/T 32918.4's KDF-degenerate rule
  applied to the agreement output. Harmless (probability 2^‚àí8¬∑klen for honest
  runs) and non-standard-but-stricter; documented here so the divergence is
  on record (Fable-5 review S7).
- **Q1.8 Failure modes** ‚Äî every fallible step returns the single
  `Error::Failed` (off-curve peer R, identity peer key, RNG failure, bad tag,
  `U = O`, bad `klen`/`id` ‚Äî indistinguishable by design; the workspace
  failure-mode invariant).
- **Q1.9 Assurance** ‚Äî new cfg-gated dudect target `ct_sm2_key_exchange`
  (class-split by static `d`, gate `|tau| < 0.20`), new fuzz target
  `fuzz_sm2_kx` (adversarial peer `R`/`S` bytes, no-panic invariant), and new
  `sm2-key-exchange` legs in the clippy/deny/MSRV/wasm32/dudect matrices.
- **Q1.10 Versioning** ‚Äî additive minor: workspace `1.0.1 ‚Üí 1.1.0`, lockstep
  sibling pins `=1.1.0`, publish posture = maintainer's call (branch + PR is
  the agent path).
