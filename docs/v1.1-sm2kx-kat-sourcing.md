# v1.1 SM2 Key-Exchange KAT Sourcing Decision

**Decision:** the normative KAT for `sm2::key_exchange` is the **GB/T
32918.5-2017 / GM/T 0003.5 recommended-curve key-exchange worked example**
(`klen = 128` bits, `K = 6C893473 54DE2484 C60B4AB1 FDE4C6E5`),
maintainer-verified against the rendered standard pages
(`/private/tmp/sm2pdf/params-hi-0{1,2}.png`, 2026-06-10).

**ID finding (Task 1.5 diagnosis):** the recommended-curve worked example
uses the **default ID `1234567812345678` for BOTH parties** — NOT
`ALICE123@YAHOO.COM`/`BILL456@YAHOO.COM` (those identities belong to the
GB/T 32918.3 *test-curve* Annex tradition and were carried over by the
plan's Task 0.2 sketch). With the ALICE/BILL IDs every point
(P_A/P_B/R_A/R_B/U) still reproduces but Z_A/Z_B — and hence K — diverge
(observed wrong K = `3C136283 0B075A6F 891DBF65 1997BF50`). `compute_z`
itself is byte-correct (pinned transitively by the sign KAT's standard
`r`/`s` and gmssl interop); the in-crate probe confirmed
`compute_z(P_A, "1234567812345678") = 3B85A571…` and
`compute_z(P_B, "1234567812345678") = 79C988D6…`, exactly the page's
KDF-input Z values. The fix was fixture-only — no library change.

## Why not gmssl CLI interop

**gmssl 3.1.1 has no SM2 key-exchange CLI** — verified: the SM2 surface is
only `sm2keygen` / `sm2sign` / `sm2verify` / `sm2encrypt` / `sm2decrypt`.
So, exactly as v0.8 SM4-CCM (gmssl lacked `-ccm`; OpenSSL EVP was the oracle —
`docs/v0.8-ccm-kat-sourcing.md`) and v0.12 SM4-XTS (gmssl lacked XTS), the KAT
is **standard-as-source**, not CLI-interop-sourced.

## Why the recommended-curve example, not the GB/T 32918.3 Annex

The GB/T 32918.3-2016 Annex A worked example runs on the `8542D69E…` **test
curve** (p, a, b, G all different from the production SM2 curve). This library
hard-codes the **recommended curve** (`p = FFFFFFFE FFFFFFFF…`,
`G_x = 32C4AE2C…`) in `sm2/curve.rs`, so the test-curve vectors **cannot
reproduce here** (the Fable-5 adversarial-review headline catch, amendment M1).
GB/T 32918.5-2017 republishes the worked example on the recommended curve;
that is the only Annex example this library can be byte-identical to.

## Transcription discipline

- `d_A` / `d_B` / `K` are filled from the recommended-curve example (confident,
  multiple independent sources agree).
- The ephemerals `r_A` / `r_B` and the confirmation tags `S_A` / `S_B` are
  **human-gated**: transcribed from the recommended-curve source and
  cross-verified (gmsm/GmSSL test code) before being trusted — never invented.
  libsm's KX test is round-trip-only and may not expose `K`/`S` constants.
- The random round-trip property test (both sides derive equal `K`, both
  confirmations pass) is the self-consistency check independent of any
  transcribed vector.
