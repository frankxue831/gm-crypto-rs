# v1.8 — TLCP certificate-pair / chain verification — Implementation Plan

> **For agentic workers:** TDD task-by-task. Steps use `- [ ]`. Each task =
> red → green → refactor → commit. `cargo test --workspace` (NOT
> `--all-targets`). Per-feature clippy passes (see CLAUDE.md). MSRV 1.85 (no
> `is_multiple_of`).

**Goal:** Add TLCP (GB/T 38636 §4-profile) certificate-pair / chain
*signature* verification — a generic single-linear-chain walk + keyUsage /
basicConstraints readers in `x509`, and the TLCP [sign, enc] pair profile in
`tlcp::chain` — per `docs/v1.8-scope.md`.

**Architecture:** Pure-core, no new dep, no_std. `x509` gains `KeyUsage` /
`BasicConstraints` readers (BIT STRING / SEQUENCE decode over the
already-captured `extensions_raw` span), `MAX_CHAIN_DEPTH`, and
`verify_chain(chain, anchors, Option<X509Time>) -> bool` (edge-signature +
Name-link walk, intermediate-CA check, unknown-critical-extension reject,
optional time window, try-all-anchors at the top). `tlcp::chain::verify_pair`
adds the role keyUsage bits, leaf-not-CA, and the non-empty/equal-subject +
equal-issuer pair binding, then calls `verify_chain` on each leg. Single
`bool` throughout. Public-inputs-only → **no dudect**; one new
`fuzz_x509_chain`.

**Tech Stack:** Rust 2024, `gmcrypto-core` (`x509` + `tlcp` features),
existing `asn1::{reader, oid}`, `spki::decode`, `sm2::verify_with_id`.

```rust
/// X.509 keyUsage (RFC 5280 §4.2.1.3) as a bitfield. A public BIT STRING
/// read — NOT a trust decision (which bits a role requires is decided by
/// `verify_chain` / `tlcp::chain::verify_pair`).
#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub struct KeyUsage { bits: u16 }

impl KeyUsage {
    /// Parse the DER `BIT STRING` (the extnValue content of a keyUsage
    /// extension). `None` on malformed input.
    fn parse(bit_string_tlv: &[u8]) -> Option<Self> {
        let (unused, value, rest) = reader::read_bit_string(bit_string_tlv)?;
        if !rest.is_empty() || unused > 7 { return None; }
        let mut bits = 0u16;
        // bit i = MSB-first within the value bytes (DER BIT STRING order).
        for i in 0u16..9 {
            let (byte, off) = ((i / 8) as usize, 7 - (i % 8));
            if byte < value.len() && (value[byte] >> off) & 1 == 1 {
                bits |= 1 << i;
            }
        }
        Some(Self { bits })
    }
    const fn has(self, i: u16) -> bool { self.bits & (1 << i) != 0 }
    #[must_use] pub const fn digital_signature(self)  -> bool { self.has(0) }
    #[must_use] pub const fn content_commitment(self) -> bool { self.has(1) } // nonRepudiation
    #[must_use] pub const fn key_encipherment(self)   -> bool { self.has(2) }
    #[must_use] pub const fn data_encipherment(self)  -> bool { self.has(3) }
    #[must_use] pub const fn key_agreement(self)      -> bool { self.has(4) }
    #[must_use] pub const fn key_cert_sign(self)      -> bool { self.has(5) }
    #[must_use] pub const fn crl_sign(self)           -> bool { self.has(6) }
    #[must_use] pub const fn encipher_only(self)      -> bool { self.has(7) }
    #[must_use] pub const fn decipher_only(self)      -> bool { self.has(8) }
}
```

```rust
/// X.509 basicConstraints (RFC 5280 §4.2.1.9): CA flag + optional
/// pathLenConstraint. `path_len` is PARSED but NOT enforced by
/// `verify_chain` (v1.8 uses `MAX_CHAIN_DEPTH` — scope D-4).
#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub struct BasicConstraints { pub is_ca: bool, pub path_len: Option<u32> }

impl BasicConstraints {
    fn parse(seq_tlv: &[u8]) -> Option<Self> {
        let (content, rest) = reader::read_sequence(seq_tlv)?;
        if !rest.is_empty() { return None; }
        let (is_ca, after) = match reader::read_tlv(content, TAG_BOOLEAN) {
            Some((b, r)) => { if b.len() != 1 { return None; } (b[0] != 0, r) }
            None => (false, content),
        };
        let path_len = if after.is_empty() { None } else {
            let (int, r) = reader::read_integer(after)?;
            if !r.is_empty() || int.len() > 4 { return None; }
            let mut v = 0u32;
            for &byte in int { v = (v << 8) | u32::from(byte); }
            Some(v)
        };
        Some(Self { is_ca, path_len })
    }
}
```

```rust
/// Find extension `extn_id`; return (critical, extnValue-content). `None`
/// if absent. Assumes from_der already shape-checked; defensive on parse.
fn find_extension<'a>(ext_tlv: &'a [u8], extn_id: &[u8]) -> Option<(bool, &'a [u8])> {
    let (seq, _) = reader::read_sequence(ext_tlv)?;
    let mut exts = seq;
    while !exts.is_empty() {
        let (ext, r) = reader::read_sequence(exts)?;
        let (oid_bytes, after_oid) = reader::read_oid(ext)?;
        let (critical, after_bool) = match reader::read_tlv(after_oid, TAG_BOOLEAN) {
            Some((b, rb)) => (b == [0xFF] || (b.len() == 1 && b[0] != 0), rb),
            None => (false, after_oid),
        };
        let (value, _) = reader::read_octet_string(after_bool)?;
        if oid_bytes == extn_id { return Some((critical, value)); }
        exts = r;
    }
    None
}

/// True iff any extension is critical AND its extnID ∉ `known` (RFC 5280
/// §4.2 — refuse a critical constraint we don't process; scope Q8.7b).
fn has_unknown_critical(ext_tlv: &[u8], known: &[&[u8]]) -> bool {
    let Some((seq, _)) = reader::read_sequence(ext_tlv) else { return false };
    let mut exts = seq;
    while !exts.is_empty() {
        let Some((ext, r)) = reader::read_sequence(exts) else { return false };
        let Some((oid_bytes, after_oid)) = reader::read_oid(ext) else { return false };
        let (critical, after_bool) = match reader::read_tlv(after_oid, TAG_BOOLEAN) {
            Some((b, rb)) => (b.len() == 1 && b[0] != 0, rb),
            None => (false, after_oid),
        };
        if reader::read_octet_string(after_bool).is_none() { return false; }
        if critical && !known.iter().any(|k| *k == oid_bytes) { return true; }
        exts = r;
    }
    false
}

impl Certificate {
    /// The keyUsage extension if present + parseable, else `None`.
    #[must_use] pub fn key_usage(&self) -> Option<KeyUsage> {
        let (_critical, v) = find_extension(self.extensions.as_deref()?, oid::KEY_USAGE)?;
        KeyUsage::parse(v)
    }
    /// The basicConstraints extension if present + parseable, else `None`.
    #[must_use] pub fn basic_constraints(&self) -> Option<BasicConstraints> {
        let (_critical, v) = find_extension(self.extensions.as_deref()?, oid::BASIC_CONSTRAINTS)?;
        BasicConstraints::parse(v)
    }
}
```
*Note:* `find_extension` returns the FIRST match (RFC 5280 forbids
duplicate extensions; from_der doesn't dedup — acceptable, a dup-extension
cert is malformed and the first wins deterministically).

```rust
/// Max total certs (leaf + intermediates) in a chain. A DoS / over-
/// restriction guard — NOT a `pathLenConstraint` substitute (scope D-4).
pub const MAX_CHAIN_DEPTH: usize = 8;

/// The extensions this profile PROCESSES (Q8.7b: a critical extension whose
/// OID is not here is refused).
const KNOWN_EXTS: &[&[u8]] = &[oid::KEY_USAGE, oid::BASIC_CONSTRAINTS];

fn within_window(cert: &Certificate, at: Option<X509Time>) -> bool {
    match at { None => true, Some(t) => cert.not_before <= t && t <= cert.not_after }
}
fn is_ca_issuer(cert: &Certificate) -> bool {
    cert.basic_constraints().is_some_and(|bc| bc.is_ca)
        && cert.key_usage().is_some_and(KeyUsage::key_cert_sign)
}

/// Verify a single linear cert chain links to a caller-trusted anchor.
/// `chain` is leaf-first in issuing order (chain[0]=leaf, chain[1..]=
/// intermediates); `anchors` are certs the caller DECLARES trusted; `at_time`
/// (Some) enforces the validity window on every cert incl. the matched anchor.
///
/// `true` iff: 1..=MAX_CHAIN_DEPTH certs; no cert carries an unknown critical
/// extension; every edge's Name links AND SM2 signature verifies; every
/// intermediate is a CA (CA=TRUE + keyCertSign); the topmost issuer is a
/// trusted anchor (Name match + signature — every same-Name anchor tried);
/// (if Some) every cert within its window.
///
/// ⚠ NOT validation, NOT endpoint authentication, NO leaf-role check (that
/// is `tlcp::chain::verify_pair`'s). The anchor's OWN signature is never
/// re-verified (trust-store model). See module docs.
#[must_use]
pub fn verify_chain(chain: &[Certificate], anchors: &[Certificate], at_time: Option<X509Time>) -> bool {
    if chain.is_empty() || chain.len() > MAX_CHAIN_DEPTH { return false; }
    for cert in chain {
        if cert.extensions.as_deref().is_some_and(|e| has_unknown_critical(e, KNOWN_EXTS)) { return false; }
        if !within_window(cert, at_time) { return false; }
    }
    // intermediate edges: chain[i] issued by chain[i+1]; chain[i+1] must be a CA.
    for i in 0..chain.len() - 1 {
        let (subj, iss) = (&chain[i], &chain[i + 1]);
        if subj.issuer_raw() != iss.subject_raw() { return false; }
        if !subj.verify_signature(&iss.subject_public_key()) { return false; }
        if !is_ca_issuer(iss) { return false; }
    }
    // top edge: chain.last() issued by a trusted anchor (try all same-Name).
    let top = &chain[chain.len() - 1];
    anchors.iter().any(|a| {
        a.subject_raw() == top.issuer_raw()
            && within_window(a, at_time)
            && top.verify_signature(&a.subject_public_key())
    })
}
```
*Bounded by construction:* one linear pass over `chain` (≤ `MAX_CHAIN_DEPTH`)
+ one pass over `anchors`. No recursion, no path-building → no cyclic-Name
hang (the reason the caller-ordered-chain contract exists).

```rust
//! TLCP (GB/T 38636 §4) certificate-PAIR verification — the [signature,
//! encryption] double-cert profile over the generic `x509::verify_chain`.
//!
//! **NOT endpoint authentication.** `verify_pair` returning `true` means the
//! sign and enc chains each link to a caller-trusted anchor, each leaf is
//! usable for its TLCP role, neither is a CA, and the two leaves share one
//! identity (non-empty byte-equal subject + byte-equal issuer). It does NOT
//! mean "this is the server I dialed" — endpoint identity binding is the
//! caller's, permanently (compare `Certificate::subject_raw`). See
//! `docs/tlcp-decomposition.md` §4.
extern crate alloc;
use crate::asn1::reader;
use crate::x509::{Certificate, KeyUsage, X509Time, verify_chain};

fn leaf_is_ca(c: &Certificate) -> bool {
    c.basic_constraints().is_some_and(|bc| bc.is_ca)
}
fn empty_subject(c: &Certificate) -> bool {
    // subject_raw is a validated SEQUENCE TLV; empty Name == empty content.
    reader::read_sequence(c.subject_raw()).is_none_or(|(content, _)| content.is_empty())
}

/// Verify a TLCP server's [sign, enc] certificate pair.
///
/// `sign_chain` / `enc_chain` are each leaf-first; `anchors` the trust set;
/// `at_time` the optional validity-window comparison. Single `bool` — never
/// a reason (the failure-mode invariant).
#[must_use]
pub fn verify_pair(
    sign_chain: &[Certificate],
    enc_chain: &[Certificate],
    anchors: &[Certificate],
    at_time: Option<X509Time>,
) -> bool {
    let (Some(s), Some(e)) = (sign_chain.first(), enc_chain.first()) else { return false; };
    // role keyUsage (keyUsage MUST be present → None ⇒ reject). The sign
    // check is a method-ref (clippy redundant_closure); the enc one is an OR.
    if !s.key_usage().is_some_and(KeyUsage::digital_signature) { return false; }
    if !e.key_usage().is_some_and(|k| k.key_encipherment() || k.key_agreement()) { return false; }
    // leaves must not be CAs.
    if leaf_is_ca(s) || leaf_is_ca(e) { return false; }
    // pair binding: non-empty + equal subject + equal issuer Name.
    if empty_subject(s) || s.subject_raw() != e.subject_raw() || s.issuer_raw() != e.issuer_raw() {
        return false;
    }
    // S1 (W2 review): equal issuer NAME does not pin the issuer KEY — require
    // the two legs to present the SAME issuing chain (byte-equal tbs from
    // index 1 up), pinning both leaves to one actual CA cert. (Vacuous when
    // both legs are length-1 — the documented same-DN-anchor residual.)
    if sign_chain.len() != enc_chain.len()
        || sign_chain[1..]
            .iter()
            .zip(&enc_chain[1..])
            .any(|(a, b)| a.tbs_raw() != b.tbs_raw())
    {
        return false;
    }
    verify_chain(sign_chain, anchors, at_time) && verify_chain(enc_chain, anchors, at_time)
}
```
