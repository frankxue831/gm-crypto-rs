## 1. Context & motivation

`gm-crypto-rs` is a published **v1.0.1** pure-Rust SM2/SM3/SM4 SDK (no_std core +
C FFI + SIMD backend). The "what's next" decision was made by triangulating a Codex
consult against a market scan:

- **The SM2/SM3/SM4 primitive space is saturated** (RustCrypto/sm2, libsm, gmsm,
  smcrypto, gm-rs) and **SM9/ZUC are already served** by existing Rust crates — so the
  opening is *not* another primitive.
- **The real gap is the trust dimension** (no existing GM crate is audited / has
  CT-verified assembly) and **the protocol/cert layer** (TLCP has *zero* pure-Rust
  implementations; everyone shells out to OpenSSL/Tongsuo).
- **SM2 key exchange (GM/T 0003.3)** is the highest *fit-adjusted* next brick: Codex
  rated it "excellent fit, medium effort." It **completes the SM2 family** (we ship
  GM/T 0003.2 sign + 0003.4/.5 encrypt; 0003.3 key exchange is the missing third),
  **reuses nearly all existing assets**, stays pure-core/no_std, and is the
  **prerequisite for an eventual TLCP capability** (TLCP `ECDHE_SM4_*` suites need it).
  X.509-with-SM2 is the natural *follow-on* (v1.2 candidate), deferred here because its
  path-validation scope creep doesn't fit a "small, auditable" cycle.

### Locked decisions (from brainstorming)

| # | Decision | Choice |
|---|---|---|
| 1 | Direction | **SM2 key exchange (GM/T 0003.3)** for v1.1 |
| 2 | Key confirmation | **Included** — full authenticated AKE (`S_A`/`S_B` SM3 tags) |
| 3 | API shape | **Role state-machines** (`Sm2KxInitiator` / `Sm2KxResponder`) |
| 4 | Packaging | **Opt-in feature `sm2-key-exchange`** (default-off, pure-core) |

## 2. Standard & protocol

Implement **GM/T 0003.3-2012 ≡ GB/T 32918.3-2016** SM2 key agreement, **with key
confirmation**. SM2 cofactor `h = 1`. Let `n` be the curve order, `G` the generator.

**Associated value (avf):** with `w = ⌈⌈log₂ n⌉ / 2⌉ − 1 = 127` for SM2,
`x̄ = avf(x) = 2¹²⁷ + (x mod 2¹²⁷)` (i.e. force bit 127 set, keep low 127 bits).

**Parties:** A = initiator, B = responder. Each holds a static key pair
(`d_A,P_A` / `d_B,P_B`), the peer's static public key, both identity strings, and the
party hashes `Z_A = compute_z(P_A, id_A)`, `Z_B = compute_z(P_B, id_B)`. `klen` = desired
key length in bytes.

**Flow (3 messages):**

```
A:  r_A ∈ [1,n−1];  R_A = [r_A]G                         ──R_A──▶
B:  r_B ∈ [1,n−1];  R_B = [r_B]G
    x̄₂ = avf(x_{R_B});  t_B = (d_B + x̄₂·r_B) mod n
    x̄₁ = avf(x_{R_A});  assert R_A on curve
    V = [h·t_B](P_A + [x̄₁]R_A);  assert V ≠ O
    K_B = KDF(x_V ‖ y_V ‖ Z_A ‖ Z_B, klen)
    S_B = Hash(0x02 ‖ y_V ‖ Hash(x_V ‖ Z_A ‖ Z_B ‖ x_{R_A} ‖ y_{R_A} ‖ x_{R_B} ‖ y_{R_B}))
    (also precompute S_2 with prefix 0x03 for verifying A's S_A)
                                                          ◀─R_B,S_B──
A:  x̄₁ = avf(x_{R_A});  t_A = (d_A + x̄₁·r_A) mod n
    x̄₂ = avf(x_{R_B});  assert R_B on curve
    U = [h·t_A](P_B + [x̄₂]R_B);  assert U ≠ O
    K_A = KDF(x_U ‖ y_U ‖ Z_A ‖ Z_B, klen)            # K_A == K_B since U == V
    verify  S_B == Hash(0x02 ‖ y_U ‖ inner)            # inner = Hash(x_U ‖ Z_A ‖ Z_B ‖ x1 ‖ y1 ‖ x2 ‖ y2)
    S_A = Hash(0x03 ‖ y_U ‖ inner)                      ──S_A──▶
B:  verify  S_A == S_2
```

Both sides agree on `K` because `U = [h·t_A](P_B + [x̄₂]R_B) = [h·t_B](P_A + [x̄₁]R_A) = V`.

## 3. Module layout

New file **`crates/gmcrypto-core/src/sm2/key_exchange.rs`**, behind a new default-off
feature `sm2-key-exchange = []` (the same pure-core `[]` gate style as `sm4-xts` —
**no new dep**). Wired
into `sm2/mod.rs` under `#[cfg(feature = "sm2-key-exchange")]`. The **default-features
build stays byte-identical** (new module is dead code unless the feature + an API call
are present).

### Reused assets (verified present, unchanged)

| Asset | Location | Use |
|---|---|---|
| `compute_z(public, id) -> [u8;32]` | `sm2/sign.rs` (pub) | `Z_A` / `Z_B` |
| `kdf(z, &mut out)` | `sm2/encrypt.rs` (`pub(super)`, sibling-visible) | `K = KDF(x_U‖y_U‖Z_A‖Z_B)` |
| `point_on_curve`, `projective_from_affine` | `sm2/encrypt.rs` (`pub(crate)`) | validate peer `R`, build points |
| `mul_g`, `mul_var` | `sm2/scalar_mul.rs` (doc-hidden pub) | `[r]G`, `[h·t](…)`, `[x̄]R` |
| fixed-budget masked nonce sampler | `sm2/sign.rs` | ephemeral `r ∈ [1,n−1]`, constant-time |
| `TryCryptoRng` bound | workspace pattern | fallible RNG → `Error::Failed`, never panic |
| `MAX_ID_LEN` guard | `sm2/sign.rs` | reject over-long `id` (via `compute_z`) |
| `FixedScalarRng` test helper | `sm2/encrypt.rs` + `sm2/sign.rs` (tests) | deterministic-ephemeral KAT |

> or needs a narrow visibility widening (same pattern as the v0.22 doc-hidden curve

## 4. API design (role state-machines)

Mirrors the repo's streaming-cipher idiom (`Sm4GcmEncryptor`: methods advance state,
the final step consumes `self`). The ephemeral secret `r` lives in `ZeroizeOnDrop`
state; advancing consumes the prior state so an ephemeral **cannot be reused** and the
**key cannot be fetched before confirmation**.

```rust
// ---- Initiator (party A) ----
pub struct Sm2KxInitiator { /* d_A, P_B, Z_A, Z_B, klen */ }
pub struct Sm2KxInitiatorWaiting { /* + r_A (ZeroizeOnDrop), R_A */ }

impl Sm2KxInitiator {
    /// `klen` and id lengths validated here; over-long id → Err(Failed).
    pub fn new(d_a: &Sm2PrivateKey, p_b: &Sm2PublicKey,
               id_a: &[u8], id_b: &[u8], klen: usize) -> Result<Self>;
    /// Sample r_A, compute R_A. RNG failure → Err(Failed).
    pub fn produce_ephemeral<R: TryCryptoRng>(self, rng: &mut R)
        -> Result<(Sm2KxEphemeralPoint, Sm2KxInitiatorWaiting)>;
}
impl Sm2KxInitiatorWaiting {
    /// Verify S_B (constant-time), derive K, emit S_A. Any failure → Err(Failed).
    pub fn confirm(self, r_b: &Sm2KxEphemeralPoint, s_b: &Sm2KxConfirm)
        -> Result<(Sm2SharedKey, Sm2KxConfirm)>;   // (K, S_A)
}

// ---- Responder (party B) ----
pub struct Sm2KxResponder { /* d_B, P_A, Z_A, Z_B, klen */ }
pub struct Sm2KxResponderWaiting { /* + S_2 expected tag, K (held) */ }

impl Sm2KxResponder {
    pub fn new(d_b: &Sm2PrivateKey, p_a: &Sm2PublicKey,
               id_a: &[u8], id_b: &[u8], klen: usize) -> Result<Self>;
    /// Sample r_B, validate R_A, derive K, emit (R_B, S_B). Holds K until finish().
    pub fn respond<R: TryCryptoRng>(self, r_a: &Sm2KxEphemeralPoint, rng: &mut R)
        -> Result<(Sm2KxEphemeralPoint, Sm2KxConfirm, Sm2KxResponderWaiting)>;
}
impl Sm2KxResponderWaiting {
    /// Verify S_A (constant-time); only then release K.
    pub fn finish(self, s_a: &Sm2KxConfirm) -> Result<Sm2SharedKey>;
}
```

Supporting types:
- `Sm2KxEphemeralPoint` — the on-wire `R` (uncompressed SEC1 `04‖X‖Y`, reusing the
  existing public-key encoding helpers). Caller-constructible from bytes → validated.
- `Sm2KxConfirm` — a 32-byte SM3 tag (`S_A`/`S_B`); compared with `ConstantTimeEq`.
- `Sm2SharedKey` — `klen`-byte secret, `ZeroizeOnDrop`, exposes `as_bytes()`.

All errors collapse to the single workspace `Error::Failed` (failure-mode invariant —
**no distinguishing** off-curve vs. bad-tag vs. RNG-fail; that would be an oracle).

## 5. Constant-time & security invariants

- Ephemeral `r` via the fixed-budget masked sampler — **no secret-dependent branch/loop**.
- `t = (d + x̄·r) mod n` and the scalar mults via `mul_var` — **no `if`/`==`/`bool` on
  secrets**; selection via `subtle`.
- `avf` reduction is pure bit-masking (set bit 127, mask low 127) — already constant-time;
  **must not branch** on `x`.
- Peer `R` validated with `point_on_curve` **before** use (invalid-curve / small-subgroup
  defense — the same guard `sm2::decrypt` uses); reject identity and `U/V = O`.
- Confirmation tags compared via `ConstantTimeEq` only.
- **Zeroize:** `r`, `t`, `(1)`-style intermediates, `x_U‖y_U`, and the KDF input buffer;
  `Sm2SharedKey` is `ZeroizeOnDrop`.
- `id.len()` bounded by `MAX_ID_LEN` (inherited via `compute_z`).

## 6. Testing & KAT

**gmssl 3.1.1 has no SM2 key-exchange CLI** (verified: only `sm2keygen/sign/verify/
encrypt/decrypt`), so — exactly as v0.8 CCM and v0.12 XTS — KAT is **not** sourced from
gmssl. A `docs/v1.1-sm2kx-kat-sourcing.md` records this decision.

- **Primary KAT:** the **GB/T 32918.5-2017 / GM/T 0003.5 recommended-curve**
  key-exchange worked example (`ALICE123@YAHOO.COM` / `BILL456@YAHOO.COM`, fixed
  `r_A`/`r_B`; `K = 6C893473…`) driven through `FixedScalarRng` — asserts `R_A`,
  `R_B`, `K`, `S_B`, `S_A` **byte-for-byte**. **Must be the recommended-curve
  (`p = FFFFFFFE…`) example, NOT the GB/T 32918.3 `8542D69E…` test-curve Annex**
  (the test-curve vectors cannot reproduce on this library — Fable-5 review catch).
- **Cross-check:** against `libsm`'s published key-exchange vectors.
- **Round-trip property:** random `(d_A, d_B)` → both sides derive equal `K`, both
  confirmations pass.
- **Negative tests:** off-curve `R` → `Failed`; flipped `S_B`/`S_A` byte → `Failed`;
  over-long `id` → `Failed`; identity/`O` results → `Failed`.

## 7. CI / dudect / fuzz

- New matrix leg `sm2-key-exchange` across **clippy**, **cargo-deny** (no new dep — pure
  allowlist no-op), **MSRV-1.85**, **wasm32** (`--no-default-features --features
  sm2-key-exchange`), and the **dudect** PR/nightly matrices.
- New cfg-gated dudect target **`ct_sm2_key_exchange`** — class-split by static private
  key `d`; both classes valid so both reach the confirmation compare via **identical
  control flow**; gate `|tau| < 0.20`. (Add the row to the CLAUDE.md dudect table; the
  base/feature counts shift accordingly.)
- New nightly fuzz target **`fuzz_sm2_kx`** — adversarial peer `R`/`S` bytes → no panic /
  no OOM / no hang (failure-mode invariant). Added to the `fuzz-nightly.yml` sweep + the
  `fuzz-build.yml` PR build.
- `cargo-public-api` baseline + `cargo-semver-checks`: new surface is **additive** behind
  a feature → passes as non-breaking; regenerate the `--all-features` baseline.

## 8. Versioning & publish posture

Additive minor → **v1.1.0**. Workspace `1.0.1 → 1.1.0`; three crates bump in **lockstep**
with exact sibling pins (`=1.1.0`), even though **`gmcrypto-c` is unchanged this cycle**
(C FFI for SM2 KX is deferred to v1.2 per the core-in-vN / FFI-in-vN+1 cadence). This is a
**publishable** cycle (new capability) — but `cargo publish` + the SSH-signed tag + the
GitHub release are the **maintainer's authenticated call**, not the agent's. The agent
path is **branch + PR** only.

## 11. Open questions / risks

- **avf width:** ✅ **resolved** — `w = 127` and the byte-indexing confirmed correct by the
  Fable-5 review (plan Task 1.1 pins it; `Fn::new` is an identity reduction since `2^128 < n`).
- **Masked-sampler visibility:** ✅ **resolved** — `sample_nonzero_scalar` is already
  `pub(crate)`; reachable from the sibling `key_exchange` module with no widening.
- **KAT curve (added by review):** the worked example MUST be the **recommended-curve**
  (GB/T 32918.5) one, not the GB/T 32918.3 `8542D69E…` test-curve Annex — see §6.
- **Confirmation-tag misuse:** document that omitting `confirm`/`finish` (not calling
  them) forfeits authentication — but the typestate makes the *key* unreachable without
  them, which is the mitigation.
- **libsm vector availability:** if libsm's vectors aren't byte-extractable, the Annex
  worked example is the sole KAT (acceptable — it's the normative source).
