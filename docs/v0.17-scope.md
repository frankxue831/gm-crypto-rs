## 1. Context

The crate source has been public on crates.io since v0.5; only the
*repository* (git history, CI, dev process) is private. A standing
open-source-readiness audit (`docs/pre-opensource-audit.md`, started v0.9,
dry-run at v0.12) already staged the entire flip and confirmed the surface is
ready: Apache-2.0 LICENSE, SECURITY.md (GitHub-advisory disclosure channel),
CONTRIBUTING.md, CODE_OF_CONDUCT.md, CODEOWNERS, issue/PR templates,
Dependabot, and a clean `gitleaks` pass over full history (0 real secrets;
noreply-only commit email; no home paths in tracked source). This cycle
**executes** that staged checklist; it does not rebuild it.

The deciding judgement: for a cryptography library, auditability is the whole
value proposition (Kerckhoffs's principle) — and the source is already
downloadable from crates.io, so staying private buys no real secrecy while
forgoing the credibility of an inspectable repo. Going public on the **0.x
line** (rather than cutting 1.0 simultaneously) gets public eyes on the code
*before* committing to the 1.0 SemVer-stability promise.

## 2. What this cycle is — and is not

**Is:** get CI off the self-hosted Mac (so the runner can be retired — a
self-hosted runner on a public repo is remote code execution), trim the
self-hosted runbook from `CLAUDE.md`, and flip repo visibility + enable the
public-only GitHub protections.

**Is not:** a crate release (no `cargo publish`, no `v0.17.0` tag — the
milestone lives in docs/CLAUDE.md/Notion only); a code change (the three
published crates are byte-unchanged); the dudect-gate hardening or fuzzing
expansion (→ v0.18); v1.0.

## 3. Q17.x — design decisions (sign-off)

- **Q17.1 — Milestone.** Public flip is **v0.17 on the 0.x line**, *not* v1.0.
  v1.0 reserved for a readiness pass (dudect-gate hardening + API-stability
  review). *(User: "v0.17 flip, 1.0 later".)*
- **Q17.2 — Not a release.** No version bump (workspace stays `0.16.0`), no
  `cargo publish`, **no signed `v0.17.0` tag** (avoids a tag that reads as a
  crates.io release; codex). crates.io skips `0.17.0` per the v0.14
  precedent.
- **Q17.3 — Scope = flip-only.** Execute the staged checklist + flip; defer
  dudect-gate hardening + fuzzing expansion to v0.18. *(User: "Flip only".)*
- **Q17.4 — CI runner = GitHub-hosted, stay there.** `ci.yml` → `macos-14`
  (same aarch64 platform: NEON SIMD + C-ABI coverage); `fuzz-nightly.yml` →
  `ubuntu-latest`. The dudect workflows **do not move** — their `|tau|`
  gates are calibrated to the `ubuntu-24.04` noise floor
  (`docs/v0.5-dudect-recalibration.md`). *(User: "Fix billing, stay
  GitHub-hosted".)*
- **Q17.5 — fuzz-nightly moves too (C1 gap).** The audit's C1 staged only
  `ci.yml`; `fuzz-nightly.yml` also ran self-hosted. It has no
  `pull_request` trigger so it was never a fork-PR RCE vector, but it must
  move so the runner can be fully decommissioned (else the nightly dies) and
  to take the adversarial fuzz workload off the personal Mac. → `ubuntu-latest`
  (libFuzzer-native; targets are platform-agnostic; 1× billing).
- **Q17.6 — Runbook removed, not relocated.** Delete `CLAUDE.md`'s
  `## Self-hosted CI runner setup` section (audit S3 default). It survives in
  git history (a throwaway `ghrunner` service account's paths, no secrets);
  no history rewrite (audit posture).
- **Q17.7 — Visibility flip is the LAST step**, gated on W1–W3 merged +
  green hosted CI + runner decommissioned. The hosted `fuzz-nightly` is
  smoke-dispatched *before* decommission (PR CI never runs the nightly).
- **Q17.8 — Branch protection** is set at the flip with the **exact
  post-swap check context names** (incl. every wasm32 matrix leg);
  solo-maintainer-friendly (block force-push/deletion, require CI checks,
  admin bypass on, no review requirement).

## 5. Deferred → v0.18 / v1.0

- **v0.18 (Harden):** dudect-gate hardening (pin the dudect-job toolchain;
  self-calibrating relative gate + multi-run median/percentile robustness —

  `ct_fn_invert`/`ct_fp_invert` from telemetry to authoritative) + round-trip

- **v1.0:** API-stability review + the v0.18 hardening → declare and publish
  `1.0.0`.
