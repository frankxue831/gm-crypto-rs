# v1.9 implementation plan — TLCP toolkit C FFI

> **For agentic workers:** TDD task-by-task. Each task: failing test → minimal
> impl → green → commit. Steps use `- [ ]`. Scope: `docs/v1.9-scope.md`
> (Q9.1–Q9.8, four forks maintainer-locked, Codex consult folded).

**Goal:** Expose the accumulated in-core TLCP toolkit (v1.6 key schedule +
no-confirm SM2-KX, v1.7 record protection, v1.8 chain/pair verification)
through the `gmcrypto-c` C ABI — a thin shim + one thin non-crypto core
adapter — so a C client can run a full TLCP handshake end-to-end.

**Architecture:** ~19 new `extern "C"` symbols + 2 opaque key-carrier handles +
~7 `#define` consts in `gmcrypto-c`, reusing the v1.2 KX handles + `CallbackRng`
and the v1.4 cert handle + time struct verbatim. One **additive** core change:
`#[doc(hidden)] pub` `verify_chain_refs` / `verify_pair_refs` (`&[&Certificate]`)
so the FFI can pass an array of cert-handle pointers without fabricating an
unsafe slice (`Certificate` is not `Clone`).

**Tech stack:** Rust 2024 / MSRV 1.85, `gmcrypto-c` (`unsafe_code = "warn"`,
cbindgen), `gmcrypto-core` (`forbid(unsafe)`), `getrandom::SysRng`,
`rand_core::TryCryptoRng`.

## Task 0 — core refs adapter (`verify_chain_refs` / `verify_pair_refs`)

**Files:** `crates/gmcrypto-core/src/x509.rs`,
`crates/gmcrypto-core/src/tlcp/chain.rs`. Tests: the existing
`x509::v1_8_tests` + `tlcp::chain::tests` (unchanged — they must stay green
through the refactor, proving byte-for-byte equivalence).

Rationale: the C `*const *const Certificate` array is not a contiguous
`&[Certificate]`, and `Certificate` is not `Clone` (Codex consult #3). Core
gains a refs-taking canonical impl; the slice-taking public fns delegate.

- [ ] **Step 1 — add `verify_chain_refs`, make `verify_chain` a wrapper.** In
  `x509.rs`, move the `verify_chain` body into:

  ```rust
  /// Reference-slice form of [`verify_chain`] — the canonical implementation.
  ///
  /// Not public API and not SemVer-covered: it exists so the C FFI shim
  /// (`gmcrypto-c`) can verify an array of cert *handles* without fabricating
  /// a contiguous `&[Certificate]` (`Certificate` is not `Clone`). Behaviour
  /// is identical to [`verify_chain`].
  #[doc(hidden)]
  #[must_use]
  pub fn verify_chain_refs(
      chain: &[&Certificate],
      anchors: &[&Certificate],
      at_time: Option<X509Time>,
  ) -> bool {
      if chain.is_empty() || chain.len() > MAX_CHAIN_DEPTH {
          return false;
      }
      for &cert in chain {
          if cert
              .extensions
              .as_deref()
              .is_some_and(|e| has_unknown_critical(e, KNOWN_EXTS))
          {
              return false;
          }
          if !within_window(cert, at_time) {
              return false;
          }
      }
      for i in 0..chain.len() - 1 {
          let (subj, iss) = (chain[i], chain[i + 1]);
          if subj.issuer_raw() != iss.subject_raw() {
              return false;
          }
          if !subj.verify_signature(&iss.subject_public_key()) {
              return false;
          }
          if !is_ca_issuer(iss) {
              return false;
          }
      }
      let top = chain[chain.len() - 1];
      anchors.iter().any(|a| {
          a.subject_raw() == top.issuer_raw()
              && within_window(a, at_time)
              && top.verify_signature(&a.subject_public_key())
      })
  }

  #[must_use]
  pub fn verify_chain(
      chain: &[Certificate],
      anchors: &[Certificate],
      at_time: Option<X509Time>,
  ) -> bool {
      let c: alloc::vec::Vec<&Certificate> = chain.iter().collect();
      let a: alloc::vec::Vec<&Certificate> = anchors.iter().collect();
      verify_chain_refs(&c, &a, at_time)
  }
  ```

  (Keep `verify_chain`'s existing doc comment on the wrapper.)

- [ ] **Step 2 — add `verify_pair_refs`, make `verify_pair` a wrapper.** In
  `tlcp/chain.rs`, import `verify_chain_refs`; move the body into
  `#[doc(hidden)] pub fn verify_pair_refs(sign_chain: &[&Certificate],
  enc_chain: &[&Certificate], anchors: &[&Certificate], at_time:
  Option<X509Time>) -> bool` (the leaves bind via `let (Some(&s), Some(&e)) =
  (sign_chain.first(), enc_chain.first())`; the S1 `zip` over `[1..]` is
  unchanged — `a.tbs_raw()` auto-derefs through `&&Certificate`; the final line
  calls `verify_chain_refs`). Reduce `verify_pair` to collect three
  `Vec<&Certificate>` and delegate.

- [ ] **Step 3 — run the v1.8 suites** (the equivalence proof):
  `cargo test -p gmcrypto-core --features tlcp,x509 x509:: ` and `tlcp::chain`.
  Expected: PASS (unchanged behaviour).

- [ ] **Step 4 — clippy** the two feature legs (`x509`; `tlcp,x509`).

- [ ] **Step 5 — commit.** `feat(x509,tlcp): refs-slice verify_chain_refs /
  verify_pair_refs for FFI (additive, doc-hidden)`.

- [ ] **Step 1 — the array adapter** (private helper):
  ```rust
  /// Build a `Vec<&Certificate>` from a C array of cert-handle pointers.
  /// `(NULL, 0)` → empty; `(NULL, n>0)` or any NULL element → None (misuse).
  unsafe fn collect_cert_refs<'a>(
      arr: *const *const gmcrypto_x509_certificate_t,
      n: usize,
  ) -> Option<alloc::vec::Vec<&'a Certificate>> {
      if n == 0 {
          return Some(alloc::vec::Vec::new());
      }
      if arr.is_null() {
          return None;
      }
      let ptrs = unsafe { core::slice::from_raw_parts(arr, n) };
      let mut v = alloc::vec::Vec::with_capacity(n);
      for &p in ptrs {
          if p.is_null() {
              return None;
          }
          v.push(&unsafe { &*p }.inner);
      }
      Some(v)
  }
  ```
